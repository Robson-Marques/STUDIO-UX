## Regras destes estudos · Rules for these studies

**PT**

- **Nada aqui é definitivo.** Um estudo é uma proposta para calibrar e aprovar a linguagem, não uma decisão. O que for aprovado orienta a Fase 2; o que for descartado **sai** (docs vivos, `STUDIO_UX.md` §9).
- **Todos derivam do `STUDIO_UX_VISUAL_DNA.md`.** Cada estudo é uma leitura do DNA — calma, sobriedade, foco no dado, nada fora da família (P20). Um estudo que contraria o DNA é inválido por construção.
- **Sem valores estéticos finais.** Como toda a Fase 1.5, os estudos exploram *caráter e composição*, não fixam paleta/px/fonte. Se um estudo precisar sugerir um valor para ilustrar, marque-o explicitamente como **provisório/descartável**.
- **Tech-agnóstico (§13).** Estudos descritivos — wireframes em prosa, composições, variações descritas — nunca código, framework ou CSS.
- **Auditável pela `STUDIO_UX_CERTIFICATION.md`.** Mesmo sendo rascunho, um estudo deve poder passar pelas dimensões da certificação (hierarquia, superfícies, ritmo, acessibilidade).
- **Cada estudo declara o que testa.** Ex.: "variação A da Sidebar — testa densidade vs. respiração"; "dashboard executivo — testa a pirâmide invertida com 4 KPIs".

**EN**

- **Nothing here is definitive.** A study is a proposal to calibrate and approve the language, not a decision. What is approved guides Phase 2; what is discarded **leaves** (living docs, `STUDIO_UX.md` §9).
- **Everything derives from `STUDIO_UX_VISUAL_DNA.md`.** Each study is a reading of the DNA — calm, sobriety, data focus, nothing outside the family (P20). A study contradicting the DNA is invalid by construction.
- **No final aesthetic values.** Like all of Phase 1.5, studies explore *character and composition*, not palette/px/font. If a study must suggest a value to illustrate, mark it explicitly **provisional/disposable**.
- **Tech-agnostic (§13).** Descriptive studies — prose wireframes, compositions, described variations — never code, framework or CSS.
- **Auditable by `STUDIO_UX_CERTIFICATION.md`.** Even as a draft, a study should pass the certification dimensions (hierarchy, surfaces, rhythm, accessibility).
- **Each study declares what it tests.** E.g. "Sidebar variation A — tests density vs. breathing"; "executive dashboard — tests the inverted pyramid with 4 KPIs".
